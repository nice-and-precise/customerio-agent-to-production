How a run flows

The same four steps the CLI, CI, and this page all share. It maps to the job: alpha-to-production ownership, evals, and observability across the fleet.

1 · agent

Provider-agnostic call (Claude first, any OpenAI-compatible endpoint second). Prompt is versioned in agent/prompts/.

2 · parse

Pulls the first balanced JSON object out of real model messiness, code fences, prose, half-formatted output.

3 · eval

Ten weighted, deterministic checks. Same scoring in the CLI, in CI, and here. A schema break or PII leak is weighted heavier than length.

4 · trace

One JSONL line per run: latency, tokens, eval score, status. Greppable today, streams to Cloud Logging / BigQuery when the fleet grows.
